{%- comment -%} SLSA v1.2 threats list — the threat taxonomy with its mitigations. Categories come from definitions/slsa_threat_categories.yml, threats from definitions/slsa_threats.yml. Both are a verbatim mirror of the SLSA spec and carry NO mitigation data. The mitigation list is DERIVED: each community record declares the threats it mitigates (`threats: [{id, status, note}]`), and we reverse-lookup that here. This is the same derived-relationship pattern personas/problems already use — a project claims its coverage on its own record, and the threat page reads it back. Nobody has to edit the static SLSA mirror to claim a mitigation. {%- endcomment -%} {%- include community-records.html -%}
